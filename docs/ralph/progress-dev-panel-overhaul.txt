--- 2026-02-09 — Feature 1.3: Cascading cleanup on room deletion ---
Status: DONE
Changed:
  - apps/pidro_server/lib/pidro_server_web/live/dev/game_list_live.ex
  - apps/pidro_server/lib/pidro_server_web/live/dev/game_detail_live.ex

What was done:
- GameListLive.handle_single_delete: added BotManager.stop_all_bots(room_code)
  and GameSupervisor.stop_game(room_code) before RoomManager.close_room.
- GameListLive.handle_bulk_delete: same cascading cleanup for each finished
  room before closing it.
- GameDetailLive.handle_event("assign_seat"): added BotManager.list_bots
  check — if a bot occupies the target position, calls BotManager.stop_bot
  before reassigning the seat.
- Added GameSupervisor alias to GameListLive imports.

Why this was prioritized:
- Functional safety — deleting a room without stopping bots/game leaves
  orphaned processes (bot GenServers, game process) that leak memory and
  can cause errors on subsequent room creation with the same code.
- Reassigning a bot's seat without stopping it left the bot process running
  against a seat it no longer owns, causing ghost actions.

--- 2026-02-09 — Feature 2.2: Quick-Create Presets (2H+2B + 4 Bots) ---
Status: DONE
Changed: apps/pidro_server/lib/pidro_server_web/live/dev/game_list_live.ex

What was done:
- Added "2H + 2B" preset button: creates room with dev host at north, starts
  bots at specifically east and west positions (using new start_bots_at_positions
  helper), leaves south open for a second human player.
- Added "4 Bots" preset button: creates room, clears host from north via
  dev_set_position(code, :north, nil), then calls start_bots(code, 4, :random)
  to fill all 4 seats with bots. This avoids the race condition described in the
  PRD — clearing the host first ensures all 4 seats are available before bots
  start joining.
- Added start_bots_at_positions/3 private helper that calls BotManager.start_bot
  for each specific position (unlike start_bots which fills first-N available).
  Includes try/catch for :noproc errors when BotManager isn't running.
- Both presets call BotManager.stop_all_bots on failure for cleanup.
- Both presets navigate to the detail view after creation.
- Buttons styled consistently: blue for 2H+2B, purple for 4 Bots.

PRD deviation note:
- The PRD suggested using dev_set_position to pre-assign bot IDs before starting
  processes. Investigation revealed this doesn't work — BotPlayer.init calls
  join_room which returns {:error, :already_seated} if the ID is already in the
  positions map. Instead, for 4 Bots: we clear the host seat, then start_bots
  fills all 4 empty positions normally.
